# Execution Policy

- Start the task through `scripts/project-state.py`.
- Modify only the current task's Allowed Files.
- Preserve unknown pre-existing user changes.
- Do not implement later tasks or unrelated improvements.
- Do not weaken tests, validation, task contracts, or policies to obtain a pass.
- Use the selected repository stack and package manager.
- Record only validation commands that were actually executed.
- Do not commit, push, or create a Pull Request unless the user or active workflow explicitly authorizes it.
