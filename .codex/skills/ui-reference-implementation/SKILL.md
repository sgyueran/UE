# UI Reference Implementation

Use this skill when implementing from a reference site, screenshot, recording, or interaction description.

## Workflow

1. Inventory visible regions, hierarchy, and navigation.
2. Separate structural ideas from brand-specific expression.
3. Map each interaction to trigger, feedback, duration, and mobile fallback.
4. Select only patterns that improve portfolio comprehension.
5. Implement an original version using repository tokens and components.
6. Compare the result at mobile, tablet, and desktop widths.

## Guardrails

- Do not reproduce proprietary logos, copy, assets, or distinctive branded artwork.
- Do not perform pixel-for-pixel cloning.
- Do not add motion that obscures responsibilities or technical depth.
- Keep page transitions under 0.9 seconds.
- Prefer screenshots or recordings as evidence; document assumptions.
- Do not add a library when CSS or an existing dependency is sufficient.

Read `references/decomposition-template.md` when analyzing a new reference.
