# Do Not Break

## Product identity

Never turn the site into:

- a generic SaaS landing page;
- a fake Unreal Editor;
- a cyberpunk neon dashboard;
- an animation showcase with weak evidence.

## Content

Never invent:

- projects;
- clients;
- employment;
- dates;
- awards;
- metrics;
- project outcomes.

Use `TODO(USER_INPUT)`.

## Navigation

Never:

- hide contact;
- hide resume;
- require animation to reach a page;
- remove keyboard or Escape behavior;
- make essential navigation gesture-only.

## Motion

Never:

- autoplay audio;
- force a long intro;
- hijack scrolling;
- ignore reduced motion;
- use animation to hide performance problems;
- add motion without purpose.

## Visual consistency

Never:

- add arbitrary accent colors;
- use random font families;
- turn every block into glass;
- mix unrelated animation styles;
- replace project evidence with decorative effects.

## Engineering

Never:

- load Three.js in the homepage initial bundle without an approved contract;
- ship heavy media without fallback;
- weaken validation to pass;
- implement future tasks early;
- edit product-authority documents during unrelated UI work.

## Review question

Before adding anything, ask:

> Does this improve role clarity, project evidence, navigation, or emotional pacing?

If not, remove it.
