# Content Strategy

## Core narrative

```text
I build and optimize real-time Unreal Engine systems.
Here is the problem.
Here is how I reasoned.
Here is what I built.
Here is the evidence.
```

## Hero

Required:

- clear UE5 role;
- main specialization;
- concise differentiator;
- selected work CTA;
- resume or contact CTA.

Recommended structure:

```text
Unreal Engine 5 Engineer

I build performant gameplay and real-time systems,
from architecture and tools to profiling and delivery.

[View selected work] [Resume]
```

Avoid:

- “Welcome to my portfolio”;
- “I love creating amazing experiences”;
- vague passion statements;
- generic web-developer language.

## Selected work

Each project should answer:

1. What was the problem?
2. What was the candidate responsible for?
3. What constraints mattered?
4. How was the system designed?
5. Which decisions were made?
6. What was optimized?
7. What outcome is verified?
8. What was learned?

## Project titles

Prefer specific system or result names:

- Modular Gameplay Ability Framework
- Runtime Procedural Environment System
- Niagara Feedback Pipeline
- UE5 Digital Twin Interaction Layer
- Lumen Performance Optimization Study

Do not invent project titles for work that does not exist.

## Project card

Recommended fields:

- project title;
- one-line problem;
- role or ownership;
- strongest media;
- Unreal domains;
- verified outcome or constraint.

Avoid:

- long tag clouds;
- fictional metrics;
- vague adjectives;
- fake client names.

## Project detail narrative

```text
Overview
→ Problem
→ Constraints
→ My Role
→ System Design
→ Implementation
→ Optimization
→ Outcome
→ Reflection
```

## About

The About page should explain:

- engineering approach;
- preferred UE domains;
- production mindset;
- collaboration style;
- current role direction.

Do not repeat the resume line by line.

## Skills

Organize by working capability:

- Gameplay & Systems
- Rendering & Optimization
- Tools & Pipelines
- Technical Art
- C++ / Blueprints
- Production & Debugging

Each group should connect to project evidence.

## Resume

The resume must be:

- clearly labeled;
- easy to download;
- available without animation;
- available from navigation and contact areas.

## Bilingual content

When Chinese and English are both used:

- write both intentionally;
- do not mix languages in one sentence;
- keep technical terms consistent;
- review translated project descriptions manually.

## Integrity

Never invent:

- projects;
- clients;
- employers;
- dates;
- awards;
- responsibilities;
- team sizes;
- metrics;
- shipped status.
