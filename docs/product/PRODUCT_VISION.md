# Product Vision

## Mission

Create a UE5 engineer portfolio that lets recruiters, lead engineers, gameplay programmers, rendering engineers, and technical artists understand within seconds:

1. who the candidate is;
2. which Unreal Engine problems they solve;
3. which projects prove it;
4. how to contact them.

## Positioning

This is not:

- a generic frontend portfolio;
- a literal Unreal Editor clone;
- a dashboard full of cards;
- a motion demo that hides weak content.

It should feel:

- cinematic;
- technical;
- controlled;
- spatial;
- credible;
- high-performance.

## Reference translation

Use the reference experience's rhythm:

```text
Orientation
→ Invitation
→ Exploration
→ Response
→ Evidence
→ Rest
```

Translate this into UE5 language through:

- viewport depth;
- camera and scene continuity;
- restrained world-grid details;
- Blueprint relationship diagrams;
- material-layer thinking;
- real project media;
- profiling and optimization evidence.

## Primary audience

### Recruiter / HR

Needs:

- role clarity;
- strongest projects;
- location and availability;
- resume and contact access.

Time budget:

- 10–30 seconds before deciding whether to continue.

### Lead Engineer

Needs:

- system architecture;
- ownership boundaries;
- debugging process;
- optimization evidence;
- trade-off thinking;
- maintainability.

### Gameplay / Systems Engineer

Needs:

- event and state architecture;
- extensibility;
- gameplay framework evidence;
- tools and production workflow.

### Technical Artist / Rendering Team

Needs:

- materials;
- Niagara;
- shaders;
- PCG;
- lighting;
- profiling;
- pipeline awareness.

## Success criteria

### In 3 seconds

The visitor knows:

- this is a UE5 engineer;
- what the main specialization is.

### In 8 seconds

The visitor sees:

- a project worth opening;
- a clear primary action;
- a route to resume or contact.

### In 20 seconds

The visitor understands:

- the candidate solves real-time engineering problems;
- the work is credible;
- there is evidence behind the claims.

## Product promises

The site must be:

- memorable without being confusing;
- technical without becoming a fake editor;
- cinematic without hiding content;
- interactive without blocking navigation;
- high-performance before decorative;
- based only on verified work.

## Non-goals

- social platform;
- job board;
- autoplay motion showcase;
- decorative 3D in every section;
- fictional project metrics;
- imitation of the reference site's protected visual expression.
