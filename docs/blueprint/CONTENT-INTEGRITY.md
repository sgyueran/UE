# Portfolio Content Integrity

## Non-negotiable rules

- Never fabricate employment, clients, team size, dates, responsibilities, performance metrics, awards, downloads, users, revenue, or project outcomes.
- Missing information must use `TODO(USER_INPUT)` in source data and must not appear as a polished claim in the rendered UI.
- Separate facts from editorial summaries.
- Quantitative claims require a provenance note in source data.
- Sample projects must be visibly marked as sample data and excluded from production release.
- Resume, email, social, demo, and repository links must be verified before release.

## Recommended content shape

Each project should distinguish: overview, role, responsibilities, UE systems, challenge, solution, outcome, media, platform, duration, team context, and verification/provenance notes.
