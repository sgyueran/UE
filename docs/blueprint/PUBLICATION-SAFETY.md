# Publication Safety

Every profile field, project, link, code sample, and media asset must have one publication status:

- `public`: verified and intentionally publishable.
- `anonymized`: publishable only after identifying details are removed.
- `permission_required`: blocked until written permission is recorded.
- `private`: never included in production output.

## Release blockers

- NDA or unreleased project information.
- Client names, logos, internal tool names, or screenshots without permission.
- Private repositories, access tokens, API keys, local absolute paths, or internal URLs.
- Commercial source code or proprietary Blueprint graphs.
- Third-party models, music, fonts, footage, or marketplace assets without display rights.
- Editor screenshots containing account names, machine paths, emails, chats, or confidential tabs.
- Unverified performance percentages, team sizes, dates, responsibilities, awards, or employment claims.

## Contact policy

Default to verified `mailto:`, GitHub, recruitment profile, and resume download links. Do not render a non-functional contact form.
