# OpenCode Fix Review Findings

Use this command only after a PR has concrete review findings.

Instructions:

- Read the unresolved PR comments and requested changes.
- Fix only the files required by the current task or review finding.
- Preserve unrelated user changes.
- Never push directly to `main` or `develop`.
- Push changes to the PR branch only.
- Run the relevant validation commands and report each result.
- Do not merge the PR.

If a requested fix is outside the active task scope, leave a PR comment explaining the scope conflict instead of changing unrelated files.
